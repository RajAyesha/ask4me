# Security Policy

## Reporting a Vulnerability

请不要在公开 Issue 中报告安全问题。

推荐方式：

1. 使用 GitHub 的 Security Advisories（Private vulnerability report / Draft advisory）
2. 如果仓库未开启安全报告入口，请改为通过维护者提供的私密联系方式报告

提交报告时请尽量包含：

- 影响范围与风险说明（可利用性、数据泄露、RCE 等）
- 复现步骤或 PoC
- 受影响版本/提交范围
- 可能的修复建议（如有）

我们会尽快确认并评估影响，在修复发布后再公开披露细节。
